fuelGroups <- function(v) {
    tmp <- as.character(v$fuelType)
    v$fuelGroup <- "Other"
    v$fuelGroup[grep("Regular", tmp)] <- "Regular"
    v$fuelGroup[grep("Premium", tmp)] <- "Premium"
    v$fuelGroup[grep("Diesel", tmp)] <- "Diesel"
    v$fuelGroup[grep("Electricity", tmp)] <- "Electricity"
    v$fuelGroup[grep("E85", tmp)] <- "E85"
    v
}

Exercise 4

Data cleaning as the distributed data objects are created is very important, because it is difficult to clean the data afterwards, or as part of the analysis. With experience, with working with the same data for the long term, it is easier to have developed a set of cleaning functions that are applied as new data is entered into storage.

How the data is divided affects the ease of different analyses. For simple tasks like calculating statistics on a single variable, these are pretty simple to achieve regardless of how the data is split. Splitting on categorical variables is effecicient if the analysis is always going to be analyzed separately for these subsets. Fitting models like glm, where all of the variables are needed together creates more problems. A simple one observed in exercise 2 was that when categorical variables have unbalanced groups some groups may not be represented in some data objects, which introduces problems when combining fits that have or are missing that level of the factor.
