Plot description: The minimum temperature is lower in the winter months of Jan, Feb, Mar, Nov, Dec, and higher in the summer months. Lows range from -30 to 40 F in the winter, and 30 to 80 F in the summer. There is more variation in the minimum values in the winter months. There is a strange data value of about 5 F recorded one July. This has to be a mistake. It is so much lower than other values, and such a very low value that it is not believable that an event like this occurred.

Couple of comments on solutions:

  • Note that there seemed to be problems trying to read the data directly off the web site. So that it was necessary to download a copy and work locally.
  • Unzgip'ing and using grep to find the row numbers is a good idea - we will get to unix tools for handling large files later in the semester
  • There is a data file containing just Iowa TMIN values, which is much smaller. It would be interesting to compare the values from both files. I initially started with the state data but was repeatedly getting problems with the data, errors in the values, so I switched back to the full data set.
  • Some groups found the minimum of the minimum values for each month and plotted these, instead of all of the daily mins. Seasonal pattern is stronger when this is done.
